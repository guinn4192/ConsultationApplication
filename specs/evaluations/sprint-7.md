## 発見したバグ

### Bug 1: 履歴画面表示中に相談画面（chat-container）も重ねて見える

- **再現手順**:
  1. オンボーディング完了後、相談画面で 1 回以上相談を送信
  2. ヘッダ「過去の相談履歴」リンクをクリック
  3. `#/history` に遷移後、画面下部を確認
- **期待動作**: 履歴画面のみ表示、相談画面（チャット履歴・モード/カテゴリ選択・送信フォーム）は非表示
- **実際の動作**: 履歴画面の下に相談画面の内容（過去メッセージ、mode/category/送信ボタン）が重ねて見える
- **原因**: `main.js` の `showHistoryListOnly()` で `chatContainer.hidden = true` を設定しているが、CSS `.chat-container { display: flex; }` が HTML の `hidden` 属性を上書きしている（`hidden` は `display: none` を意味するが、class セレクタの `display: flex` の方が specificity で勝つ）
- **修正案**: CSS に `.chat-container[hidden] { display: none; }` を追加、または `style.css` の `.chat-container` ルールを `:not([hidden])` で修飾
- **重大度**: Medium（機能はすべて動作するが、ユーザーが履歴画面から相談画面の内容を同時に見てしまい紛らわしい）
- **影響範囲**: Feature 20 の履歴画面全体。Feature 21 の再開モーダル表示時はモーダルが overlay として上に乗るため視覚的影響は小さい

## 改善提案

1. **Bug 1 の修正**: CSS に `.chat-container[hidden] { display: none !important; }` を 1 行追加すれば解決
2. **履歴画面のページネーション**: SPEC には明記されていないが、セッション数が増えると一覧が長大になるため将来的に必要
3. **F20 詳細画面の戻る UX**: 「← 履歴一覧に戻る」と「← 相談画面に戻る」が両方存在するが、ユーザーが迷う可能性あり（今回は両方とも動作確認済み）
4. **サーバ起動時の orphan auto-close**: 今回は実機検証していないが、DESIGN §7.7 に規定済み。Sprint 7 以降の運用で前日セッションが自動 close されていることをモニタリング推奨
